# Security Policy

## Reporting a Vulnerability

We take the security of our deterministic runtime and signed contracts very seriously. If you discover a security vulnerability within Axiom Core, please help us by disclosing it responsibly.

**Please do not report security vulnerabilities via public GitHub issues.**

Instead, please send an email to **contact@yashmakan.com**.

### What to include:
- A description of the vulnerability.
- Steps to reproduce (or a proof-of-concept).
- The potential impact of the vulnerability.

### Our Commitment:
- We will acknowledge receipt of your report within 24 hours.
- We will provide an estimated timeframe for a fix.
- We will notify you once the vulnerability has been patched.

We thank you for your help in keeping Axiom Core secure for all users.
